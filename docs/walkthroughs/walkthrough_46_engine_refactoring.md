# エンジン リファクタリングおよびサウンドエンベロープの復元 (Plan 46) [2026-07-20 08:35]

C言語シーケンサー実装の過程で発生した副作用（音色変化）の修正と、フレームバッファモジュールの責務整理（ゲームループの分離）を行いました。

## 実施内容

### 1. サウンドエンベロープ（ADSR風減衰）の再実装
- C言語側 (`sound_synth.c`) にて、音符の発音開始から0.5秒かけて音量を落とすエンベロープ（アタック〜ディケイ）を再実装しました。
- トーン波形（矩形波など）のサスティン（減衰後の維持音量）を40%に設定し、長い音符でも音が途切れないようにしつつ、以前の「ノコギリ波」のようなアタック感を復元しました。
- ノイズ波形（ドラム・ハイハットなど）のサスティンを0%に設定し、歯切れの良いパーカッション音を復元しました。

### 2. ゲームループの分離（アーキテクチャリファクタリング）
- これまで `engine/hal/framebuffer_*.py` に記述されていたウィンドウ生成やゲームループ（`run()`, `_tick()` など）を分離し、新しく以下のモジュールへ移動しました。
  - `engine/hal/engine_micropython.py`
  - `engine/hal/engine_cpython.py`
  - `engine/hal/engine_wasm.py`
- これにより、`framebuffer_*.py` は純粋に「画面バッファと描画処理」のみを扱うモジュールとなり、責務が明確化されました。

## 次のステップ
- **パフォーマンスチューニング**: サウンド処理で浮動小数点演算を使用しているため、ゲーム全体のFPS低下（10 → 7, 8）が発生しています。次回は固定小数点化による計算の軽量化と、FreeRTOSを利用したタスクのCore 0 / Core 1への分散配置（ロードバランシング）を行います。
