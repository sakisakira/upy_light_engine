# ST7789 SPI通信オーバーヘッド削減 計画

MicroPython環境（Cardputer）において、ディスプレイ転送時のSPI通信オーバーヘッドを劇的に削減し、エンジンのFPSを向上させるための最適化計画です。

## メモリ（ヒープ）への配慮と安全な設計
全画面分のRGB565バッファ（約64.8KB）を一括で確保すると、既存のINDEX8バッファ（約32.4KB）と合わせてヒープを約97KBも消費してしまい、Cardputerの内蔵SRAM環境では `MemoryError` が発生するリスクが極めて高くなります。（お客様の懸念の通りです！）

そこで、**「画面をいくつかのブロック（チャンク）に分割して転送する」** アプローチをとります。
画面の高さ（135px）は 5 で割り切れる（27px）ため、1回につき 27行分 のパレット変換と転送を行う設計にします。

- バッファサイズ: 240px × 27行 × 2byte = **12,960バイト（約12.6KB）**
- 1フレームあたりのSPI通信回数: 135回 → **5回** （96%のオーバーヘッド削減！）

これにより、メモリの安全性を完全に保ったまま、通信オーバーヘッドをほぼゼロに近いレベルまで激減させることができます。

## Proposed Changes

### [MODIFY] `engine/hal/st7789.py`
1. **バッファの初期化 (`__init__`)**
   - 既存の `self.line_buf = bytearray(self.width * 2)` を廃止します。
   - 新たに、5分割転送用のバッファ `self.chunk_buf = bytearray(self.width * 27 * 2)` （12,960バイト）を確保します。

2. **Viper関数の最適化 (`_send_lines_viper`)**
   - 1行ずつ `spi_write` を呼んでいた処理を廃止します。
   - 代わりに、渡されたバッファサイズ分（`w * h_chunk` 個のピクセル）を一次元ループで一気にパレット変換するだけの純粋な高速変換関数に書き換えます。

3. **転送処理 (`show`)**
   - 5回のループを回し、各ループ内で以下の2つを行います：
     1. `_send_lines_viper` を呼んで27行分を `chunk_buf` にパレット変換。
     2. `self.spi.write(self.chunk_buf)` で27行分を一括転送。
