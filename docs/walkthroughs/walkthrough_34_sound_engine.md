# サウンドエンジンとMMLの和音対応アップデート

MicroPythonのサウンドエンジン (`sound_engine.c`) とPython API (`sound.py`, `mml_parser.py`) をアップデートし、マルチトラック再生と複数の波形をサポートすることに成功しました。

## 変更内容

### 1. サウンドエンジン (Cモジュール)
- CardputerのI2Sオーディオ出力向けに、ハードウェアによる4和音（4チャンネル）ミキシング機能を追加しました。
- `Square`（矩形波）、`Sawtooth`（ノコギリ波）、`Triangle`（三角波）、`Noise`（ノイズ）の波形を追加しました。
- 音の切れ目を滑らかにするため、単純な線形減衰エンベロープ（リニアディケイ）を実装しました。

### 2. Python API の強化
- `sound.play_mml(mml_string)` が、カンマ `,` で区切ることで複数トラックのMMLに同時対応するようになりました。
  - 例: `sound.play_mml("T100 O4 C4, T100 O4 E4, T100 O4 G4")` と書くと、Cメジャーの和音が鳴ります。
- プリセットの効果音（例: `"jump"`, `"coin"`, `"hit"` など）を簡単に鳴らせる `sound.play_sfx(name)` を追加しました。
- ゲームループの更新をスッキリ中央集権化するため、`engine/__init__.py` に `engine.update()` を追加しました。

### 3. 検証
- 実機での再生を確認しました。
- シリアル通信の過負荷バグや、初期化漏れ（`SoundHAL.tracks`）のバグを修正しました。
- 起動時に和音のコード進行テストを鳴らし、その後に元のBGMを鳴らすよう `main.py` を更新しました。

## テスト方法
ローカルの `main.py` を変更して、Cardputer上で自由にテストしてみてください：
1. `main.py` のMMLを変更したり、`engine.sound.play_sfx("jump")` をボタンに割り当てたりしてみます。
2. `scripts\install_to_cardputer.ps1 -Port COM4` を使って、更新したスクリプトを送信します。
3. デバイスをソフトリセットまたはハードリセットすると、変更した音が聞こえます！
