# Sprite API のエンジン側実装完了！

エンジンの高レベル描画APIとして、`Sprite` クラスと `sprite()` メソッドの基礎実装が完了しました。

## 実装内容
* `engine/sprite.py`: `Image` への参照と切り出し領域、色情報を保持する `Sprite` クラスを実装しました。
* `engine/image.py`: `Image` から簡単に `Sprite` を生成できる `subimage` ヘルパーメソッドを追加しました。
* **各フレームバッファ実装**: CPython / WASM / MicroPython のそれぞれの `Framebuffer` クラスに対して `def sprite(self, cx, cy, spr, rotate=0.0, scale=1.0)` を実装しました。
  * 現在はフォールバック版としてのピュアPython実装です（ニアレストネイバー法によるアフィン変換）。
  * ご指摘いただいた通り、今回は `rotate` を実装せず (`0.0` 固定として扱い)、`scale` と **中心座標 (`cx`, `cy`) を基準とする描画** に注力しました。
* `main.py` の更新: `blt` から `sprite` メソッドを使ったテストに変更し、スケール（`scale=1.5`）が正常に機能することを確認しました。

### スケールテストの様子（`main.py`）
`main.py` 内で、スプライトが中心座標から描画され、グラデーションスプライトが 1.5倍 に拡大されて正常に描画されていることが確認できました。
