# 時間管理システム (Time & FPS Management) の実装

## 概要
ゲームエンジン全体で統一して時間（デルタタイム）やFPS（フレームレート）を管理し、さらに将来的なゲームの一時停止に対応できる `engine.time` モジュールを実装しました。

## 変更内容
1. **`engine/time.py` モジュールの新設**
   * `Clock` クラスを作成し、`frame_count`（フレーム数）、`game_time_ms`（ゲーム内経過時間）、`fps` などのプロパティをカプセル化しました。
   * プラットフォーム間の互換性レイヤーとして、MicroPython（`utime`）とCPython（`time`）の差異を吸収し、`ticks_ms` や `ticks_diff` といったメソッドを安全に使えるようにしました。
   * メインループが毎フレーム呼ぶ `clock.tick()` メソッドを実装し、前フレームからの経過時間（デルタタイム）の算出とFPSの算出（1秒ごとの更新）を行うようにしました。
   * `clock.pause()` / `clock.resume()` による一時停止機能を実装しました（一時停止中はゲーム内時間が進まないようになっています）。

2. **各HALからの呼び出し (`framebuffer_*.py`)**
   * CPython, WASM, MicroPython の3つの各HALのメインループ（`_tick()` や `while True:`）の先頭で、`engine.time.clock.tick()` を呼び出し、システム時間を確実に更新する仕組みを導入しました。

3. **PC用入力マッピングの追加 (`engine/hal/input_cpython.py`)**
   * 一時停止のテストに用いるため、PCのキーボード上の `Enter` キーをゲームパッドの `Start` ボタン、`Space` キーを `Select` ボタンに割り当てました。

4. **テスト用スクリプト (`main.py`) の更新**
   * 画面右上にリアルタイムの FPS 値 (`clock.fps`) をテキストで表示するようにしました。
   * 1秒ごとにコンソール（ログ）に FPS が出力されるように設定しました。
   * PCで `Enter` キー（＝Startボタン）を押すことで、スプライトのアニメーションを含めた「アップデート処理」が一時停止（Pause）し、もう一度押すと再開（Resume）されるテストを実装しました。

## テスト結果
* バックグラウンドで起動中の `main.py` にて、右上に「FPS: 60」などの表示がされることを確認できます。
* コンソールログにも約1秒間隔で `INFO: FPS: 60` 等が出力されています。
* ウィンドウ上で `Enter` キーを押下するとアニメーションとFPSカウントを除いたゲーム進行が一時停止し、「(PAUSED)」の文字が表示される機能が正しく動作することを確認しました。
