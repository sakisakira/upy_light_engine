# Sprite API の回転対応

## 概要
エンジンの `sprite()` メソッドにおいて、画像を中心に沿って回転(`rotate`)させながら描画する機能を実装しました。
アフィン変換による逆マッピングを用いることで、ニアレストネイバー法による回転と拡縮を同時に処理できるようにしています。

## 変更内容
1. **`engine/hal/software_renderer.py` の更新**
   * `draw_sprite` メソッド内に `rotate != 0.0` の場合の分岐を追加しました。
   * 回転角度とスケール値から、事前にアフィン変換の逆行列に相当する定数 (`cos_inv`, `sin_inv`) を計算するようにしました。
   * **バウンディングボックスの計算**:
     描画先の矩形（クリッピング範囲）を必要最小限にするため、元のスプライトの四隅を回転・拡縮してスクリーン上の最小・最大座標を求めるロジックを実装しました。
   * **描画ループの最適化**:
     Python上でのパフォーマンス低下を防ぐため、ループの最内側（X軸のイテレーション）では `sin`, `cos` などの重い関数を一切使用せず、事前計算された定数の加算と乗算、および `// 1` (切り捨て) 演算のみでピクセルのマッピング座標を導出するようにしています。

2. **`main.py` へのテスト用スプライトの追加**
   * 回転の様子が視覚的に明確にわかるように、非対称な図形である **「各頂点の色が異なる（赤・緑・青）RGB三角形」** を描画する `create_triangle_sprite` ヘルパー関数を追加しました。
   * この三角形スプライトを画面右下付近で毎フレーム回転 (`rotate=frames * 0.05`) させながら描画するように設定しました。

## テスト結果
* エンジン単体テスト `main.py` を実行し、RGB三角形の各頂点が綺麗に回転し、欠けやノイズなどのアーティファクトが発生しないことを確認しました。
* 回転と同時にスケーリング(`scale`)も並行して正しく適用されることが確認できました。
