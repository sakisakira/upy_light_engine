# Walkthrough: Engine-side Gapless Loop Support (Plan 43)

エンジン側のギャップレスループ再生の仕組み（Plan 43）の実装が完了しました。

## 概要

これまでBGMのループは「ゲームのフレームレート」に依存した擬似的なループを行っていましたが、本対応によりMMLパーサーと各プラットフォームのオーディオHAL層（`SoundHAL`）がネイティブに無限ループを管理できるようになりました。

### 実装した内容

1. **MMLパーサーの拡張 (`engine/mml_parser.py`)**
   - MML内に `$` マーカーを記述できるように拡張しました。
   - `parse_mml()` が音符のリストを1つだけ返すのではなく、`$` を境にして `(intro_tracks, loop_tracks)` の2つのリストに分割して返すようになりました。

2. **Cモジュール連携の機能強化とリネーム (`engine/hal/sound_synth_cpython.py`)**
   - ご要望通り、PC用であることを明確にするため `sound_synth.py` からリネームしました。
   - `render_wavs(intro_tracks, loop_tracks)` を実装しました。
   - ここが**今回の最大のポイント**です。オシレータの波形状態（位相）を保ったまま `intro_tracks` のWAVと `loop_tracks` のWAVを連続して生成するため、WAVの繋ぎ目における波形のズレ（プツッというノイズ）が発生しません。

3. **PC版シミュレータのループ対応 (`engine/hal/sound_cpython.py`)**
   - イントロ部分とループ部分の2つのテンポラリWAVファイルを出力します。
   - まずイントロ部分を再生し、`update()` の中で再生時間が経過したのを検知すると、即座にループ部分のWAVの無限ループ再生へと移行するロジックを実装しました。
   - （※Windowsでは `winsound.SND_LOOP` でネイティブループし、Mac/Linuxでは `update()` の中で再呼び出しを行います）

4. **WASM版およびCardputer実機版の対応 (`sound_wasm.py`, `sound_micropython.py`)**
   - ハードウェアのオシレータを直接制御できる強みを活かし、`intro_tracks` が終わったトラックから順次 `loop_tracks` に切り替え、以後無限にループするよう `update()` 内の進行ロジックをリファクタリングしました。

5. **BGMデータの再生成 (`tools/convert_bgm.py`)**
   - BGM変換ツールを修正し、生成される各トラックの先頭（初期設定直後）に `$` マーカーを自動挿入するようにしました。
   - これにより、現状の全BGMは「イントロなし（即座に全体がループする）」として振る舞います。

## テスト内容

- `convert_bgm.py` を実行し、BGM定義ファイルに `$` マーカーが正しく挿入されたことを確認。
- PC版シミュレータ (`python game/main.py`) をバックグラウンドで起動し、エラー終了せず正常にゲームとサウンドループが回っていることを確認。

> [!NOTE]
> 現在のあなたのPC環境で `python game/main.py` を起動していただくことで、BGMのループが正しく機能しているか、そして音の繋ぎ目にノイズや極端なラグがないかをお手元で確認していただけます。
