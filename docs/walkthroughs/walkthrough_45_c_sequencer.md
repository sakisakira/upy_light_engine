# C-Based UBGM Sequencer Implementation

BGMのリズムに生じていた波（揺れ）を完全に排除するため、`.ubgm` 形式の再生シーケンサーをPythonのゲームループ依存から脱却し、**ファームウェア（C言語側）のオーディオレンダリングスレッドへ完全移行**しました。

## 変更内容

### 1. C Core へのシーケンサー統合
- `c_modules/core/sound_synth.h`, `sound_synth.c` に、`.ubgm` のバイナリデータを解析し、サンプル単位の時間経過に同期して自律的に次の音符（周波数・波形・音量）をフェッチするシーケンサー処理（`_ubgm_fetch_next_note`）を実装しました。
- Python側から渡されたバイナリデータをヒープ（`malloc`）に保持し、最大4トラックの同時進行をハードウェアタイマーレベル（オーディオバッファ生成単位）の精度で制御します。

### 2. SFX 用のチャンネルオーバーライド機能
- ゲーム中の効果音（SFX）は引き続きPython側で制御するため、C側のシーケンサーが特定のチャンネル（デフォルトはチャンネル3）を無視する機能 `set_channel_override(ch, true)` を追加しました。
- これにより、BGM再生中にSFXが重なっても、CのシーケンサーがBGMの無音データで上書きしてしまうのを防ぎます。

### 3. WASM との互換性維持
- WASMの AudioWorklet (`audio_worklet.js`) に `play_ubgm` イベントを追加。Python（`sound_wasm.py`）から送られたバイナリデータをWASMヒープ領域へ `malloc` し、ポインタ渡しでCエンジンにシーケンスを実行させる処理を実装しました。
- そのため、**WASM版でも実機（Cardputer Adv）と全く同じ精度・挙動のシーケンサー**が動作します。

### 4. 不要になった処理の削除
- `sound.update()` による BGM 向けのシーケンス更新が不要になったため、Python側（`sound_micropython.py`, `sound_wasm.py`）にあった `.ubgm` のパース処理や複雑なループ管理を撤廃し、シンプルな `play_ubgm_data(data)` へリファクタリングしました。
- （※SFX 用の再生チェックは引き続き必要なため、ゲームループ内の `sound.update()` 呼び出し自体は保持しています）

## 検証結果
- WASM版にてテスト画面（`main.py`）を起動し、和音およびBGMループが正確なリズムで崩れなく再生されることを確認しました。
- Cardputer Adv 用の新しいファームウェア（ESP32）のビルドを完了し、実機でも同様の精度が実現できるようになりました。
