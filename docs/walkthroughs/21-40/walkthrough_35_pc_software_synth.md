# PC版ソフトウェアシンセサイザーの実装 (Plan 35)

PCシミュレータ上で、Cardputer実機のハードウェア音源を完全にエミュレートするソフトウェアシンセサイザー (Plan 35) の実装が完了しました。

## 変更内容

### 1. ピュアPythonシンセサイザー (`engine/hal/sound_synth.py`)
- MMLトラックを解析し、標準的な 16-bit 44.1kHz のPCM波形データを生成する独立モジュール `sound_synth.py` を新規作成しました。
- 実機で使用している `sound_engine.c` と全く同じ波形合成ロジックをエミュレートしています。
  - `Square`（矩形波）、`Sawtooth`（ノコギリ波）、`Triangle`（三角波）、`Noise`（ノイズ）の全4波形に対応（ノイズの質感も同じLCGアルゴリズムで再現）。
  - 実機と同じ0.5秒のリニアディケイ（線形減衰）エンベロープを適用。
  - ミキシング後の音割れを防ぐためのソフトクリッピング (`[-32768, 32767]`) を実装。
- メモリ上の巨大なバイト列処理の高速化のため、Python標準の `array.array('h')` を活用し、複数トラックの長い和音でも一瞬 (約0.05秒以下) でWAVバイト列を生成できるようにしました。

### 2. Windows / Mac 用 HAL の更新
- **Windows (`sound_cpython_win.py`)**: 従来の古い単音矩形波ジェネレータを削除し、`sound_synth.render_wav()` を呼び出してWAVファイルを生成する方式に変更しました。
- **Mac (`sound_cpython_mac.py`)**: 同様に、従来のジェネレータを `sound_synth.render_wav()` に置き換えました。
- どちらのモジュールも、1番目のトラックの長さだけを見るのではなく、すべての並行トラックを通しての「最大デュレーション（＋減衰の余韻時間）」を正しく計算するように修正しました。

## 確認内容
- PCシミュレータ上で `main.py` を実行しました。
- 4和音のMMLや様々な波形の効果音が、`PyAudio` などの外部ライブラリを一切インストールすることなく、OS標準の非同期オーディオ再生API (`winsound.PlaySound` / `afplay`) を通じて遅延なく完璧に再生されるようになりました。

これで、Cardputer実機と全く同じレトロなサウンドをPCシミュレータ上でも確認しながらゲーム開発を進められます！
