# 機能追加: PCエミュレータのウィンドウリサイズ対応

このプランでは、PC上（CPython）でのゲーム開発をより快適にするため、ゲーム画面（Tkinterウィンドウ）の自由なリサイズ機能を追加します。単純に引き延ばすのではなく、Pyxelや一般的なゲームエンジンのように**「アスペクト比（240x135）を維持したまま、余白（黒帯/レターボックス）を追加して中央に配置する」**機能を実現します。

## レビューが必要な項目

- リサイズへの追従は `hal/framebuffer_cpython.py` 内のTkinterイベントループで完結するため、**ゲーム開発者側（`main.py`等）のコード変更は一切不要**です。
- リサイズ時の拡縮には、ドット絵のピクセル感を損なわないよう `PILImage.NEAREST`（ニアレストネイバー法）を継続して使用します。

## 提案する変更内容

---

### `hal/framebuffer_cpython.py`
ソフトウェアレンダラー（Tkinter表示部）に、動的スケーリングと余白描画のロジックを追加します。

#### [MODIFY] `hal/framebuffer_cpython.py`
- **Tkinterのレイアウト設定変更 (`run` メソッド)**:
  - `_canvas.pack()` を `_canvas.pack(fill=tk.BOTH, expand=True)` に変更し、ユーザーのウィンドウ操作に合わせてキャンバス領域自体が常に画面いっぱいに広がるように設定します。
- **動的スケーリングとレターボックス計算 (`_tick` メソッド)**:
  - リサイズされた最新のキャンバス幅・高さを `_canvas.winfo_width()` と `_canvas.winfo_height()` で取得します。
  - 元の解像度（240x135）との比率を計算し、`scale = min(cw / 240, ch / 135)` で「画面に収まる最大のスケーリング倍率」を導き出します。
  - スケーリング倍率を用いて `PILImage` を `new_w = int(240 * scale)`, `new_h = int(135 * scale)` にリサイズします。
  - キャンバスの中央に配置するためのオフセット座標 `offset_x = (cw - new_w) // 2`, `offset_y = (ch - new_h) // 2` を計算します。
  - キャンバスに以前描画した画像を削除してから、計算したオフセット座標に新しい画像を配置することで、自動的に上下または左右に黒い余白（レターボックス）が形成されます。

---

## 検証プラン（Verification Plan）

### 手動での動作確認
1. `python main.py` を実行してゲームウィンドウを立ち上げます。
2. マウスでウィンドウの端をドラッグし、横長・縦長に自由にリサイズします。
3. リサイズに合わせてゲーム画面がピクセル感を保ったまま拡大・縮小され、かつアスペクト比（16:9）が崩れずに上下左右に黒帯（レターボックス）が正しく表示されることを目視で確認します。
