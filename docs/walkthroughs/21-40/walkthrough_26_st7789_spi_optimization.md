# ST7789 SPI通信オーバーヘッド削減 完了報告

MicroPythonのSPI転送ボトルネックを排除する最適化を実装しました。

## 実装内容
`upy_light_engine/engine/hal/st7789.py` における `show()` の転送処理を、メモリ安全性を考慮した **「5チャンク分割転送（1チャンク27行）」** に変更しました。

1. **チャンクバッファの確保**
   - 画面1行分（480バイト）のバッファに代わり、27行分（12,960バイト）のRGB565バッファ `self.chunk_buf` を初期化時に確保するように変更しました。
   - 内蔵SRAMへの負担を最低限（約12.6KB）に抑えつつ、通信回数を激減させます。
2. **SPI通信の大幅削減**
   - 以前は「135回/フレーム」発生していた `self.spi.write()` の呼び出しを、「5回/フレーム」に削減しました。これにより、Python側からハードウェアへのコンテキストスイッチによる遅延（推測：毎フレーム50〜80ms）を 96% 排除しました。
3. **Viper関数の最適化**
   - `_send_lines_viper` を、純粋な「パレット変換専用ループ」に特化させ、引数として渡す `spi_write` コールバックを削除しました。
   - `x` と `y` の二重ループを廃止し、チャンク内の全ピクセル（6,480ピクセル）を一気になめる高速な1次元ループに変更しました。
