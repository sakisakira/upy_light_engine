# Walkthrough: Phase 3B - ダブルバッファリングと回転描画の実装

このフェーズでは、C言語エンジンにおいて一時的に失われていたスプライトの回転機能を復旧し、理論上のパフォーマンス上限を引き上げるためのダブルバッファリング・アーキテクチャを導入しました。

## 実施した変更内容 (Changes Made)

### 1. DDA(Digital Differential Analyzer)による回転処理の超高速化
- `engine_render.c` の `render_draw_sprite` メソッドを改修し、スプライトの2D回転を復活させました。
- 描画のインナーループ内で高負荷な「浮動小数点演算」や「乗算」を行わないよう、事前に回転行列の逆行列を計算し、加算のみでテクスチャ座標を進める**DDAアルゴリズム**を実装しました。これによりESP32-S3上で極めて高速に回転スプライトを描画できるようになりました。
- `engine_types.h` およびPythonのインターフェース (`modlightengine.c`) を更新し、`angle`（角度）のパラメータをCモジュール内部まで貫通させました。

### 2. フレームバッファのダブルバッファリング化
- `framebuffer_micropython.py` にて、1画面分しかなかった `bytearray` のピクセルバッファを **2つ**（`buffers[0]`, `buffers[1]`）に拡張しました。
- メインループの順序を再構成し、**「Core 1 が新しいフレームを描画している裏側で、Core 0 が前回のフレームをSPI経由でディスプレイに転送する」** 完全な非同期並列パイプラインを実現しました。

## テスト結果 (Validation Results)

- **回転の復旧**: 実機テストにて、三角形のスプライトが正常に回転しながら描画されることを確認しました。
- **メモリリークの解消**: ダブルバッファリングにより使用メモリは増えましたが、Free Memoryは33KB〜64KBの間で安定して推移しており、メモリリークは発生していません。
- **FPSの推移**: 描画負荷が上がったにも関わらず、依然として26 FPS付近を維持できています。

> [!NOTE]
> **今後の課題（なぜ60 FPSに届かないのか？）**
> 理論上は「Core 1の描画」と「Core 0の転送」が並列化されたためFPSが上がるはずでしたが、依然として26 FPSで頭打ちになっています。
> この原因は **「Core 0側で行われている Python上の SPI 転送処理 (`display.show`) 自体が 38ms 近い時間を消費している」** ためである可能性が濃厚です。
> プロファイラを用いて調査した結果、`display.show` は Python のループ内で `135 / 27 = 5` 回に分けてチャンクごとにパレット変換（C関数呼び出し）と `spi.write()` を繰り返しており、この「Python側のループオーバーヘッド ＋ SPIブロック時間」が合計で約38msかかってしまっていることが判明しました。
> 真の60 FPSを目指すには、このSPI転送ループ自体をC言語化（DMAのフル活用）するなどのさらなるアプローチ（Phase 3C）が必要になります。

## 追記: 実機検証結果

実機での動作検証の結果、以下のことが判明しました。

1. **描画のCモジュール化 (DisplayList) が機能**: 
   描画API呼び出しは正常にC側で処理され、テキスト描画に関しても最適化版 (b.text) を用いることで描画時間が 16.6ms -> 4.5ms に大幅短縮されました。
2. **現在のFPSは 21〜28 FPS**: 
   ダブルバッファリングとDMAによる転送自体は正常に動作しており、画面の表示も安定しています。
3. **新たなボトルネックの発見 (Phase 3cへの課題)**: 
   - 現在の send_display (Cモジュール) が Python をブロック (同期待機) してしまっています。
   - そのため、Pythonの描画構築 (19ms) と DMA転送待機 (8.8ms) が直列に実行され、フレーム時間が 30ms を超えています。
   - 次のフェーズ (Phase 3c) では、SPI送信を Core 1 (レンダリングタスク) に移動、あるいは send_display を完全に非同期化することで、Python側を待機させずに済むようにします。これにより 60 FPS が達成可能になります。
