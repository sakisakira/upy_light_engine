"# 実装計画: 3. ボタン入力とキーマッピング対応

先ほどの `image` と `framebuffer` の整理に続き、「3. ボタン入力とキーマッピング対応」の実装プランです。ゲームを操作するための入力基盤（キーボード＆ゲームパッド対応）を整備します。

## User Review Required

実装を進めるにあたり、PC環境でのゲームパッド対応の手段や、APIの仕様についてご意見をお聞かせください。

## Open Questions

> [!NOTE]
> 1. **PC側のゲームパッド対応について:**
>    Pythonでゲームパッド（コントローラー）の入力を取る場合、`pygame` ライブラリ（のジョイスティック機能だけ）を使うのが最も手軽でクロスプラットフォームです。依存ライブラリとして `pygame` を入れてもよろしいでしょうか？（それとも、依存を増やさず Windows API の `XInput` を直接叩く等、別の方法が良いでしょうか？）
> 2. **`btnp` / `btnr` の実装:**
>    Pyxelでは「押されているか（`btn`）」だけでなく、「今押された瞬間か（`btnp`）」「今離された瞬間か（`btnr`）」の判定がよく使われます（メニュー操作など）。これも併せて実装しておいてよろしいでしょうか？
> 3. **Cardputer側（MicroPython）の入力実装:**
>    CardputerのキーボードはI2Cなどで読み取るハードウェア固有の処理が必要になるかと思いますが、現時点ではハードウェア操作用のコードが存在しないため、「関数やマッピングの枠組みだけ作っておき、キー読み取り処理の中身は後で埋められるダミーにしておく」という形で進めてもよろしいでしょうか？
