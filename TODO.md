# 実装計画

今すぐに実装して欲しい訳ではないけど、将来的にこれらを見据えて作業をして欲しいことを列挙します。
「こうした方がいいのではないか」という意見があれば言って下さい。

* HAL
  * ターゲットはCardputer Adv, PC(Windows, macOS), Web (WASM)とします。
* フレームバッファ
  * 低レイヤの `blt()` (左上基準の単純矩形転送) とは別に、ゲーム開発向けの高レベル描画APIとして `sprite()` を追加する。
    * `Image` オブジェクト自体に切り出し領域 (u, v, w, h) を持たせる仕組み（SubImage）を導入する。
    * **生成時に色変換（パレット置換やtint）を行える機能も持たせ、ゲーム側での独自画像切り出し・色変換処理をエンジン側に巻き取る。**
    * `sprite(cx, cy, img, colkey=-1, rotate=0, scale=1.0)` のように、**中心座標基準**で `u, v, w, h` を省略できるシンプルなシグネチャにする。
    * 回転角はラジアン指定、拡縮はアスペクト比維持（ニアレストネイバー）とする。
* 文字描画
  * 描画色を引数から指定できるようにする。
  * パフォーマンス（特にMicroPython環境）を考慮し、影付き文字などの重ね描きパターンはあらかじめアトラス画像側に合成・収録しておき、1回の `blt` で高速に描画できるようにする。
    * （フォント変換ツール側の拡張と、エンジン側でのスタイル/オフセット指定の仕組みを追加する）
    * 重ね描きの例は game_result.py の text() などが参考になりそう。
* 画像フォーマット
  * PNGから変換するツールも用意したい。(PC側)
    * Asepriteフォーマットから変換できると便利かも知れないがフォーマットが公開されていなければバージョンアップ追従で痛い目に会いそうなので、これはオプション。
* サウンド
  * Cardputer Advで可能なサウンド機能を付けたい。1音ぐらいなら行ける? (done)
  * MMLで音楽演奏も欲しい。 (done)
  * Cardputer Adv上で公式MicroPythonではサウンドICに直接アクセスできないのであれば、そこをブリッジするCモジュールを作成する。メンテを考慮して可能であれば薄い関数で。(done)
    * Cモジュール作成にはDockerを使って他のPCに移動させても開発を継続できるようにしたい。(done)
    * Cモジュールの動作確認ができたらM5モジュール利用部分は削除する。(done)
* ボタン入力
  * A, B, X, Y, Selectぐらいを使う想定だが、将来的に追加する可能性もあり。
  * Cardputer Adv上の好きなキーをボタンに割り当てられる機能が欲しい。
  * PC側はキーとゲームコントローラに対応できると良い。(オプション)
  * **マウスおよびタッチパネル入力のサポート**:
    * 画面上の仮想ボタン（UI）をクリック・タップで操作できるようにするため、`mouse_x`, `mouse_y`, `mouse_button` などのポインティングデバイスの座標と状態を取得できるAPIをエンジンに追加する。
      * Cardputer Advでは入力を無視する。
* テストの一時的なファイルは 「tests/tmp」 ディレクトリに置くようにして下さい。
* アサーション。リリース時に実行速度に悪影響を与えないようにしつつ、開発時には問題発覚の機会を増やしたい。

  * FPSを表示し、ログにも出力する。(done)
